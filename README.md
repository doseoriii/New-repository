# 海外项目跟进指南

一个面向海外业务团队的轻量级 Web 应用，用于 **客户追踪** 与 **项目跟进**。
每个成员用各自的账号密码登录，只能看到自己登记的内容；总管理员可以看到所有人的数据，
并且能够新增成员、修改成员密码。

## 特点

- 🔐 账号密码登录，密码使用 scrypt 加密存储
- 👤 普通成员：仅能看到 / 编辑自己登记的客户与项目
- 🛡️ 总管理员：查看全部成员数据、新增成员、修改密码、删除成员
- 🗂️ 两大模块随意切换：客户追踪、项目跟进
- 📊 管理员概览：成员数据量统计、最近登录记录
- 💾 数据保存在本地 `data.json`，零外部依赖，开箱即用

## 运行方式

需要 Node.js（已内置，无需安装任何依赖）：

```bash
node server.js
```

启动后浏览器打开： http://localhost:3000

> 默认管理员账号： **admin / admin123**
> 首次登录后请务必在「管理员面板」修改默认密码。

## 角色与权限

| 功能 | 普通成员 | 总管理员 |
| --- | --- | --- |
| 查看/编辑自己的客户、项目 | ✅ | ✅ |
| 查看其他人的数据 | ❌ | ✅ |

## 数据文件

所有数据存储在项目根目录的 `data.json` 中。请定期备份该文件。

## 部署到云平台（让别人在各自电脑登录）

本应用是纯 Node.js（零依赖），`node server.js` 即可启动，天然适配云平台，部署后把公网网址发给同事即可。

> ⚠️ **持久化提醒**：多数免费云平台的文件系统是临时的，重启/重新部署会清空 `data.json`。
> 请务必把数据目录挂到「持久化磁盘卷」，并设置环境变量 `DATA_DIR` 指向挂载路径（如 `/data`）。

### 方式 A：Railway（推荐，支持持久卷）
1. 将本项目推到 GitHub 仓库。
2. railway.app → New Project → Deploy from GitHub repo。
3. 启动命令自动识别 `npm start`（也可使用自带 `railway.json`）。
4. 在项目中添加 Volume，挂载路径填 `/data`，并设置环境变量 `DATA_DIR=/data`。
5. 部署完成获得公网网址，发给同事登录。

### 方式 B：Render
1. 推到 GitHub，在 render.com 新建 Web Service 关联仓库。
2. Build Command：`npm install`，Start Command：`node server.js`。
3. 免费版文件系统临时，**数据会随重启清空**；正式使用建议升级套餐或改用外部数据库。
